Have you found yourself checking the Dragon*Con guest list far too often? Stop wasting time, bandwidth, and sanity; have D*C Notifications do the obsessive checking for you!

D*C Notifications works by checking the structure of a Dragon*Con web page similar to the way a search engine does. The difference is that a search engine catalogs an entire site, but D*C notifications looks for specific data. Once an hour, a script looks at the list of guests on the D*C website, and compares it to the guests that it has already noted. If new guests are found notifications are sent via Twitter and email.

This method has it's disadvantages. Reading specific data from a web page requires that the data is consistently structured. If the format of the page changes, the data-fetching script will need to be updated to reflect the changes. As a result, there are no guarantees that script will correctly parse the page. D*C Notifications cannot be expected to be reliable. If you need official information on Dragon*Con, see the official Dragon*Con website.

The source code for this project is avaible on GitHub.
